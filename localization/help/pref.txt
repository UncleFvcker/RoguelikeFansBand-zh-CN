<style:title>用户首选项 (User Preferences)</style>

FrogComposband 允许你更改游戏的各个方面以适应你的喜好。你可以定义键位映射（改变按键映射到底层命令的方式），创建宏（允许你将单个按键映射为一系列按键），修改视觉效果（允许你改变怪物、物品或地形特征的外观），更改颜色（允许你使特定颜色变得更亮、更暗，甚至完全不同），或设置选项（打开或关闭它们）。

<topic:Files><style:heading>首选项文件</style>

首选项文件分为两种：系统首选项和用户首选项。前者代表默认的显示和输入选项，而后者是你为了覆盖默认设置而添加的可选自定义设置。系统首选项文件（如 pref.prf）通常存储在相对于安装目录的 /lib/pref 中（例如，在 Windows 上，如果你安装在 C:\games\FrogComposband，那么系统 pref 文件可能在 C:\games\FrogComposband\lib\pref 中）。另一方面，用户首选项文件存储在用户目录中，相对于安装位置，在 Windows 上是 /lib/user，而在 Linux 上可能是 ~/.angband/FrogComposband（这是可自定义的）。总的来说，用户文件的位置取决于平台，并且会根据你配置和安装游戏的方式而有所不同。

在初始化期间，游戏会加载系统首选项文件 pref.prf，该文件会引导加载其他首选项文件，例如用于键盘映射的 pref-key.prf 和用于默认选项设置的 pref-opt.prf。然后，会加载用于视觉效果的系统首选项文件，从用于正常 ASCII 显示的 font.prf 开始，到为那些喜欢奇怪的图形界面的玩家准备的 graf.prf。这些系统首选项文件中的每一个都会根据你正在游玩的平台重定向到另一个文件（例如在 Windows 上的 font-win.prf）。

最后，一旦你的角色出生或加载完毕，游戏将尝试加载以下用户首选项文件：
  <color:G>user.prf
  user-$SYS.prf
  $RACE.prf
  $CLASS.prf
  $PLAYER.prf</color>
在这里，$SYS 是你所在平台的 3 字符系统代码，例如 Windows 端口的 <color:keyword>win</color> 或 Curses 端口的 <color:keyword>gcu</color>。$RACE 是你当前角色的种族，$CLASS 是职业名称，$PLAYER 是角色的实际名字，除非它包含你的文件系统不允许的非法字符（例如，把你的玩家命名为 <color:keypress>Mr. *WONDER*</color> 可能不是个好主意）。用户首选项文件最后加载，以便你可以用自己的设置覆盖系统首选项。我们将在下面讨论一些示例。

首选项文件包含一个命令列表，这些命令会被解析以配置各种选项。接下来将对此进行讨论。

<topic:Commands><style:heading>首选项命令</style>

有许多用于设置首选项的命令。语法很复杂，但在大多数情况下，你可以使用用户界面命令 <color:keypress>%</color>、<color:keypress>&</color> 和 <color:keypress>@</color> 来轻松完成操作，将语法的烦恼交给游戏本身处理。在下一节中，我将给出一个语法示例，然后我们将讨论使用那些专为你隐藏语法细节而设计的用户界面命令来设置首选项。

<topic:Example><style:heading>示例</style>

在这个例子中，启动游戏并加载你的角色。我的角色被很有想象力地命名为 <color:keyword>Test</color>，所以当我稍后提到 Test.prf 时，你就会知道要将那个文件名在心里替换成你角色的名字。

在这个例子中，我们将直接输入一个单独的用户首选项命令。为此，我们使用特殊的 <color:keyword>Enter a user pref command (输入用户首选项命令)</color> 命令，该命令可通过 <color:keypress>!</color> 键使用。这需要我们知道我们想要的偏好设置命令的确切语法，以及各种值的内部数字代码。这是一个高级命令，通常你会想要使用更友好的游戏内命令（在这种情况下，是关于视觉效果的命令）来代替。

假设你觉得 @ 看起来很无聊。让我们改用 <color:y>@</color> 吧！要进行此更改，输入 <color:keypress>!</color> 然后输入 <color:keypress>R:0:11:@</color>。<color:keypress>R</color> 首选项命令用于设置怪物种族（你可能知道，它们在 /lib/edit/r_info.txt 中定义）的显示值。游戏使用怪物种族表中的特殊第 0 个条目作为玩家的显示选项。我们输入了 <color:keypress>R:0</color> 表示我们想要覆盖这个值。下一个字段是要使用的属性（颜色）。我们需要在 16 种可用颜色中输入一个 0 到 15 之间的值；我为黄色选了 11。最后，我们保留 @ 作为这个怪物种族的字符。输入此命令后，你会发现玩家仍然是一个普通的旧 @；输入 <color:keypress>^R</color> 强制重新绘制，现在你应该自豪地显示为 <color:y>@</color> 了。

如果我们此时重新启动游戏，我们的更改将会丢失。我们稍后将讨论 <color:keyword>Interact with Visuals (与视觉效果交互)</color> 命令，但现在，只需盲目地按下 <color:keypress>%</color>，然后按下 <color:keypress>1</color>，将当前的怪物种族视觉效果转储到用户首选项文件中。我得到了默认列出的 Test.prf 文件（记住，我的角色实际上叫 Test）。你应该保留这个 $NAME.prf 建议，因为我相信你还记得，这是游戏在初始化期间尝试自动加载的文件之一。保留这个值可以确保我们下次玩这个角色时自动使用我们的更改！所以按下 <color:keypress>ENTER</color> 实际转储你的偏好设置。

最后的补充：你可以直接编辑你转储的 Test.prf 文件，或者你也可以删除它。由于这只是一个例子，你可能只想把它删掉，但你应该知道，游戏将你当前角色的所有用户首选项都存储在这里（默认情况下），这也包括你可能定义的所有宏。

<topic:Visuals><style:heading>与视觉效果交互 (<color:keypress>%</color>)</style>

<topic:Colors><style:heading>与颜色交互 (<color:keypress>&</color>)</style>

此选项允许你配置游戏使用的基本颜色。这可能会有点麻烦：虽然用户界面很容易弄清楚（使用 <color:keypress>R</color>、<color:keypress>G</color> 和 <color:keypress>B</color> 键调整 RGB 值），但每次按键只会增加或减少 1，因此达到你想要的值可能需要一段时间。在这里同样，<color:G>记得转储你的更改</color>！

修改颜色可能在某些平台上不起作用。

<topic:Options><style:heading>与选项交互 (<color:keypress>=</color>)</style>

关于各种游戏选项的说明记录在其他地方（参见 <link:option.txt>）。在这里，我只提一下存在两个用于开启和关闭选项的首选项命令：你可以使用 <color:keypress>Y:option_name</color> 开启选项，使用 <color:keypress>X:option_name</color> 关闭选项（使用 <color:keypress>!</color> 调用此命令）。显然，请将 option_name 替换为你想要配置的选项的确切名称。前一个链接的文档记录了内部选项名称的列表。

<style:note>注意：<color:normal>你无法使用用户首选项命令 X 或 Y 更改作弊选项。你也无法在 <color:keyword>pref-opt.prf</color> 系统首选项文件中配置这些选项。要作弊，你需要启用 <color:keyword>allow_debug_opts</color>，然后通过用户界面设置选项。但这会将你的角色标记为作弊者！</color></style>

<topic:Macros><style:heading>与宏交互 (<color:keypress>@</color>)</style>

“与宏交互”命令允许你定义或移除<color:keyword>宏 (macros)</color>，宏是从单个逻辑按键到一系列按键的映射，允许你使用键盘上的特殊键，比如功能键或数字小键盘键，可能结合修饰键，来“自动化”你经常使用的、需要重复按下多个按键的命令。

由于宏代表按键序列，而且并非所有按键都有可打印的表示形式，因此宏的触发器和动作通常必须“编码”成人类可读的形式。这可以通过几种类型的编码来完成，包括“\xHH”代表十六进制的字符编号 HH，“\e”代表“Esc”转义码，“\n”代表“换行 (newline)”码，“\r”代表“回车 (return)”码，“\s”代表“空格 (space)”码，“\\”代表反斜杠，“\^”代表脱字符，以及“^X”代表任何“控制 (control)”键“ctrl-X”的代码。

在 FrogComposband 中，用作宏触发器的特殊按键将显示其真实名称，例如 Control 键加上 F1 键显示为“\[control-F1]”，Alt 键加上 G 显示为“\[Alt-G]”。在其他一些变体中，这些特殊按键使用键码显示。

请注意，宏的“动作”不会检查是否触发了其他宏，因此你无法制造无限循环。你可以指定非常长的宏，但宏的长度受底层输入机制的限制，该机制通常将触发器和动作限制在约 1024 个键以内。

特殊的“\”命令（在宏中必须编码为“\\”）在宏中非常有用，因为它绕过了所有的键位映射，并允许将下一个按键视为底层 FrogComposband 命令集中的命令。有关 FrogComposband 命令集的列表，请参阅“command.txt”帮助文件。例如，一个将 Shift-KP6 (小键盘6) 映射为“\”+“.”+“6”的宏将引发“向东跑”的行为，无论用户选择了哪种按键组合，也无论定义了什么键位映射。

宏可以在用户首选项文件中指定为一对文本行，一行格式为“A:<str>”，定义了编码后的宏动作；另一行格式为“P:<str>”，定义了编码后的宏触发器。

<topic:Keymaps><color:U>键位映射 (Keymaps)</color>

“与宏交互”命令还允许你定义<color:keyword>键位映射 (keymaps)</color>，这与宏有些松散的联系。键位映射将单个按键映射为一系列按键，并会绕过其他键位映射以及所有宏。FrogComposband 使用键位映射将原始 (original) 和 roguelike 按键集映射到底层命令集，并允许用户修改或添加自己的键位映射。请注意，所有的键位映射动作必须使用底层命令指定，而不是使用原始或 roguelike 按键集中的按键。原始按键集与底层命令集几乎相同，只是“数字”被映射为“;”加上一个方向，“5”被映射为“,”，以及少数几个控制键被映射为各种操作。有关完整的底层命令集，请参阅“command.txt”。键位映射的一些用途包括通过将其映射为“\x00”来“禁用”某个命令。键位映射比宏更安全，因为<color:G>宏应始终使用特殊按键作为触发器</color>；使用普通按键作为宏触发器会干扰该键的正常功能，但只要该键不覆盖命令，使用普通按键作为键位映射触发器是可以的。

键位映射可以在用户首选项文件中指定为一对文本行，格式为“A:<str>”和“C:<T>:<key>”，其中 <str> 是编码后的键位映射动作，<T> 是按键集（0 代表原始，1 代表 roguelike），<key> 是编码后的触发键。

<topic:MacroSamples><style:heading>一些宏和键位映射的示例</style>

你现在可以忘掉键位映射，假装宏是允许你（用户）将一系列命令绑定到单个按键的东西。以下是一些示例：

<color:G>示例 #1：箭术 (Archery)</color>
在你的主要弹药上铭刻 <color:v>@f1=g</color>，然后将序列 <color:keypress>f1*5</color> 或 <color:keypress>*5f1</color> 映射到你喜欢的宏按键上。这将自动瞄准最近的敌人并射出带有铭刻的弹药；并且 =g 部分的铭刻允许你在战斗结束后自动捡起弹药。（如果 <color:keyword>use_old_target (使用旧目标)</color> 选项被关闭（默认情况），则使用 <color:keypress>f1*5</color>；如果你开启了该选项，则使用 <color:keypress>*5f1</color>。）

<color:G>示例 #2：休息 (Resting)</color>
将宏序列 R&\r 添加到一个方便的键上，比如 F1。通常，休息需要 2 次击键（<color:keypress>R</color> 和 <color:keypress>Enter</color>），一些玩家在他们的休息不断被打断时会感到这种操作令人沮丧。

<color:G>示例 #3：照明 (Illumination)</color>
在你的照明法杖 (Staff of Illumination) 上铭刻 <color:v>@ug</color>，然后将序列 <color:keypress>ug</color> 映射到你喜欢的键上，比如 F6。当你找到一根照明魔棒 (Rod of Illumination) 时，只需将宏动作更改为 <color:keypress>za</color>，你就可以继续使用同一个键来照亮房间了。

<color:G>示例 #4：批量鉴定 (Mass Identify)</color>
在你的鉴定法杖 (Staff of Identify)（带 50+ 发充能的那个）上铭刻 <color:v>@ui</color>，然后将 ui* 绑定到你喜欢的知识查询键上。一旦你的背包被未鉴定的物品填满，就使用这个宏。

<color:G>示例 #5：法术 (Spells)</color>
假设你最喜欢的攻击法术是一本你已经铭刻了 <color:v>@mf</color> 的法术书里的法术 <color:keypress>e</color>。让我们用 <color:keypress>y</color> 作为这里的触发键；对于这个触发键，我们需要的是键位映射而不是宏，否则我们会干扰 <color:keypress>y</color> 键的正常功能！为 <color:keypress>y</color> 创建一个以 <color:keypress>mfe</color> 为动作的键位映射；如果你想更进一步并自动瞄准最近的怪物，可以使用 <color:keypress>mfe*5</color> 或 <color:keypress>*5mfe</color>。有关更多法术示例，请参见 <link:magic.txt#Macros>。

<color:G>示例 #6：截图 (Screenshots)</color>
截图键 <color:keypress>)</color> 有些局限性，它只在特定情况下起作用，并且可能会覆盖掉诸如消息等潜在有趣的材料。特殊键 <color:keypress>x1d</color>（键码 29）将在任何时候进行截图而不会有这样的缺点，但许多键盘或平台没有办法实际触发键码 29。将宏序列 \x1d 添加到你最喜欢的键（如 F11）上，并用它在任何你想截图的时候截图！（注意，这必须是一个宏，而不是一个键位映射。）

<style:note>切记将你的宏和键位映射追加保存到一个文件中。</style>
